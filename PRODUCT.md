# Product

## Register

product

## Users

Zenith Admin serves administrators, operators, approvers, and system maintainers who work inside a dense backend management interface. They need to configure business modules, inspect operational state, process workflow tasks, and resolve exceptions efficiently without losing trust in the data.

## Product Purpose

The product provides a full-stack admin system for managing enterprise back-office capabilities. Success means users can complete routine operations, approvals, configuration, monitoring, and audits quickly, with clear status feedback and predictable behavior across modules.

## Brand Personality

Professional, clear, restrained. The interface should feel reliable and efficient rather than decorative, with a consistent component vocabulary that lets the work stay central.

## Anti-references

Avoid over-decorated dashboard visuals, inconsistent table and form affordances, non-standard modal behavior, decorative motion, weak empty/error states, and page-specific UI patterns that diverge from the admin system conventions.

## Design Principles

- Keep workflows task-first: every page should make the next action and current status obvious.
- Preserve consistency across CRUD pages: tables, search toolbars, operation columns, modals, empty states, and refresh behavior should follow the shared system patterns.
- Surface operational risk clearly: approval, automation, trigger, and monitoring states need explicit labels and recoverable error feedback.
- Prefer trustworthy density: show enough information for administrators to act without adding ornamental complexity.
- Make failure states actionable: validation, API errors, missing configuration, and empty data should explain what happened and what the user can do next.

## Accessibility & Inclusion

Target WCAG AA contrast and keyboard-accessible interactions. Respect reduced-motion preferences, avoid relying on color alone for workflow state, and keep labels, buttons, and table operations readable in Chinese enterprise admin contexts.
