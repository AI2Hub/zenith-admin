# 指标中心（语义指标）

指标中心把「销售额」「活跃用户数」这类业务口径**定义一次、处处复用**：指标绑定数据集，声明聚合或公式、维度与显示格式，仪表盘与数据预警直接引用指标，避免同一口径在各处被重复（且不一致地）实现。在「报表中心 → 指标中心」（`/report/metrics`）管理。

## 指标类型

| 类型 | 说明 |
|------|------|
| **简单指标** | 对数据集某个字段做聚合：`sum / avg / max / min / count / distinct_count` |
| **比率指标** | 两个口径相除（如 转化率 = 下单数 / 访问数），用公式表达 |
| **复合指标** | 由其它指标/字段组合的公式计算 |

## 新建指标

| 字段 | 说明 |
|------|------|
| 指标名称 / 编码 | 编码全局唯一，创建后不可修改 |
| 指标类型 | 简单 / 比率 / 复合 |
| 数据集 | 指标的取数来源 |
| 来源字段 / 聚合方式 | 简单指标必填 |
| 公式 | 比率/复合指标必填；只允许后端安全公式语法，服务端安全求值 |
| 维度字段 | 允许按哪些字段分组下钻（逗号分隔） |
| 时间字段 | 用于按时间粒度（小时/天）聚合的列 |
| 单位 / 显示格式 | 如 `元`、`0,0.00` |
| 负责人 / 指标目录 | 归属治理（见[资源治理](./governance)） |
| 统计口径 / 说明 | 业务口径的文字定义，供使用方对齐 |

## 生命周期与发布快照

指标生命周期为 **`draft（草稿）→ published（已发布）→ deprecated（已废弃）`**：

- 编辑与生命周期操作携带 `expectedRevision` 乐观锁，并发冲突返回 409，需刷新后重试；
- 「**发布**」以当前修订固化**发布快照**，仪表盘、预警等运行端只消费发布快照——继续改草稿不影响引用方，直到再次发布；
- 「**废弃**」后引用方会看到生命周期警告，但不会阻断合法访问；
- 仅**草稿**状态的指标可删除；废弃前应先检查「引用关系」。

## 评估与引用关系

- **预览（评估）**：列表点「预览」→「执行计算」即时得到指标当前值（按显示格式与单位渲染），并显示耗时与是否命中缓存；计算由服务端安全执行，仅返回聚合结果，不下发原始查询。
- **引用关系**：查看该指标被哪些仪表盘组件、预警规则、派生指标引用，为变更与下线提供依据。

## 在其它模块中使用

- **数据预警**：预警规则的「来源类型」选**指标**即可直接监控指标值，无需重复配置聚合（见 [AI 问数与数据预警](./ai-and-alerts#数据预警)）。
- **仪表盘**：组件可绑定指标获得统一口径的取数与格式化。

## 权限

| 操作 | 权限码 |
|------|--------|
| 查看 / 选项 / 引用关系 | `report:metric:list` |
| 新增 | `report:metric:create` |
| 编辑 | `report:metric:update` |
| 删除（仅草稿） | `report:metric:delete` |
| 评估 / 预览 | `report:metric:evaluate` |
| 发布 / 废弃 | `report:metric:publish` |
